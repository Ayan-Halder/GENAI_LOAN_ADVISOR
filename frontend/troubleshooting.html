Troubleshooting

1. FastAPI Server Not Starting

If the FastAPI server does not start, check the following:

  • Ensure you have activated the virtual environment.
  • Make sure all required packages are installed. You can do this by running: pip install -r requirements.txt
  • Check for syntax errors in your Python files.

2. CORS Policy Errors

If you encounter CORS policy errors when accessing the frontend:

  • Verify that your FastAPI app has CORS middleware correctly set up.
  • Ensure that the origin of your frontend is allowed in the CORS settings.

3. Environment Variables Not Loaded

If your environment variables (e.g., GEMINI_API_KEY) are not being recognized:

  • Make sure you have a .env file in the root directory with the correct key-value pair.
  • If the .env file is not working, you can export variables directly in your terminal: 
                    # For Linux or macOS
                export GEMINI_API_KEY='your_api_key_here'

                # For Windows
                set GEMINI_API_KEY='your_api_key_here'

4. No Response from the GenAI API

If the GenAI API is not returning any response:

  • Ensure that your API key is valid and has the necessary permissions.
  • Check your internet connection.
  • Examine the server logs for any error messages.

5. Incorrect Data Formatting

If you receive unexpected results or errors related to data formatting:

  • Ensure that all input data matches the expected types (e.g., numbers for credit score, loan term, etc.).
  • Validate that the JSON structure sent to the backend is correct.

6. CSS or JavaScript Not Loading

If your styles or scripts are not applying correctly:

  • Ensure that the paths to your CSS and JavaScript files are correct.
  • Check the console for any errors related to loading resources.
  • Clear your browser cache or use incognito mode to see if changes reflect.

7. Unexpected Application Behavior

If the application behaves unexpectedly:

  • Check the browser's console for JavaScript errors.
  • Make sure the backend is running correctly and responding as expected.
  • Review your code for any logical errors or overlooked edge cases.

If you encounter issues not listed here, please consult the documentation or reach out to the project maintainers for assistance.
